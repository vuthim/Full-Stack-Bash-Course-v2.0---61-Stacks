## Reporting a Vulnerability

If you discover a security vulnerability within this course, please follow these steps:

### How to Report

1. **DO NOT** create a public GitHub Issue for security vulnerabilities
2. Email the maintainer directly (or contact via private message)
3. Include the following information:
   - Type of issue (e.g., vulnerable script, insecure pattern)
   - Full paths of relevant files
   - Step-by-step instructions to reproduce
   - Proof-of-concept or exploit code (if possible)
   - Impact assessment

### What to Expect

- **Acknowledgment**: Within 48 hours, you'll receive acknowledgment of your report
- **Update**: Within 7 days, you'll receive an update on the progress
- **Resolution**: We'll work to release a fix and credit you (unless you prefer anonymity)

### Scope

This course contains educational scripts. While we strive for best practices, scripts in this course:

- Are for **learning purposes**
- May require **root/sudo** access (clearly marked)
- Are **not production-ready** without modification

### Best Practices for Users

When using scripts from this course:

1. Always review code before running
2. Test in a safe environment first
3. Never run scripts with root privileges unless necessary
4. Keep backups before running system modification scripts

This course includes Stack 23 (Security Scripting) which covers:
